# Automation Testing for Sauce Demo using Playwright

This is an End-to-End (E2E) automation testing project for the [Sauce Demo](https://www.saucedemo.com/) website. This project is built using Playwright and follows the Page Object Model (POM) design pattern to ensure clean, maintainable, and scalable code.

## ✨ Key Features

- **Page Object Model (POM):** Separates locators and action functions from test scenarios for better readability and maintenance.
- **Authentication State:** Uses an authentication setup to log in once and run all feature tests in a logged-in state, making test execution much faster.
- **Data-Driven Testing:** Uses [Faker.js](https://fakerjs.dev/) to generate random data (names, zip codes, etc.) for more dynamic and realistic testing.
- **NPM Scripts:** Provides command shortcuts to easily run various testing scenarios.
- **Cross-Browser:** Configured to run on multiple browsers (Chromium, Firefox, WebKit) in parallel.

## 🛠️ Technology Stack

- **Framework:** [Playwright](https://playwright.dev/)
- **Language:** JavaScript
- **Dependency Management:** Node.js & NPM
- **Data Generator:** [Faker.js](https://fakerjs.dev/)

## 📋 Prerequisites

Before you begin, ensure you have the following software installed on your machine:
- [Node.js](https://nodejs.org/en/) (v16 or higher is recommended)
- NPM (usually installed with Node.js)

## 🚀 Installation & Setup

1.  **Clone this repository:**

    ```bash
    git clone [YOUR_REPOSITORY_URL]
    cd [PROJECT_FOLDER_NAME]
    ```

2.  **Install all dependencies:**
    This command will download Playwright and all other required libraries (like Faker.js).
    
    ```bash
    npm install
    ```

3.  **Install Playwright browsers:**
    This command will download the necessary browsers (Chromium, Firefox, WebKit).

    ```bash
    npx playwright install
    ```

## 📂 Project Structure

- **`/pages`**: Contains all Page Object files. Each `.js` file here represents a page or a major component of the application and contains its relevant locators and action functions.
- **`/tests`**: Contains all test scenario files (with the `.spec.js` extension).
    - `auth.setup.js`: A dedicated test to perform login and save the authentication state.
    - `*.spec.js`: Functional test files for various features.
- **`playwright.config.js`**: The main configuration file for Playwright, where projects, browsers, and the authentication workflow are defined.
- **`package.json`**: Defines project dependencies and contains the NPM scripts for running tests.
- **`user.json`**: The file generated by `auth.setup.js` containing the authentication state (cookies, local storage). This file is included in `.gitignore`.

## 🏃 Running Tests

This project uses NPM Scripts to simplify test execution. Here are the available commands:

- **Run all feature tests (headless):**
  This command runs all functional tests (except for the login tests) in a headless Chromium browser, using the existing session.

  ```bash
  npm run test:features:headless
  ```

- **Run all login tests (headed):**
  This command will open a browser and run all test scenarios found in the login.spec.js file.

  ```bash
  npm run test:login
  ```

- **Add extra arguments:**
  You can pass any Playwright flag after --. For example, to run only the tests with "Invalid" in their title:

  ```bash
  npm run test:login -- -g "Invalid"
  ```

- **Run all tests (default):**
  This command will run all projects defined in playwright.config.js.

  ```bash
  npm test
  ```

## 📊 Viewing Reports

After the test execution is complete, Playwright automatically generates an interactive HTML report. To open it, run the following command:

  ```bash
  npx playwright show-report
  ```
